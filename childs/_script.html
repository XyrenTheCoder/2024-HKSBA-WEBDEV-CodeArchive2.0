HTML <SCRIPT> Tag

Definition and Usage

The <SCRIPT> tag is being used for embedding a client side script written in JavaScript.

It is often placed in the <head> tag but it can also be placed in <body>, depending when the script needs to be loaded.

The <script> element either contains scripting statements, or it points to an external script file through the src attribute.

Attribute(s)

Attribute Value Description
async async Specifies that the script is downloaded in parallel to parsing the page, and executed as soon as it is available (before parsing completes). (Only for external scripts)
crossorigin anonymous
use-credentials		 Sets the mode of the request to an HTTP CORS Request.
defer defer Specifies that the script is downloaded in parallel to parsing the page, and executed after the page has finished parsing. (Only for external scripts)
integrity filehash Allows a browser to check the fetched script to ensure that the code is never loaded if the source has been manipulated.
nomodule boolean Specifies that the script should not be executed in browsers supporting ES2015 modules.
referrerpolicy no-referrer
no-referrer-when-downgrade
origin
origin-when-cross-origin
same-origin
strict-origin
strict-origin-when-cross-origin
unsafe-url		 Specifies which referrer information to send when fetching a script.
src URL Specifies the URL of an external script file.
type scripttype Specifies the media type of the script.

The <SCRIPT> tag supports Global Attributes in HTML.

Examples and Syntax

<!DOCTYPE html>
<html>
    <head>
        <title>web title</title>
        <script>
            document.getElementById("id").innerHTML = "hello world";
        </script>
    </head>
</html>

Default CSS Values and Settings

Most browsers display the <SCRIPT> element with the default values below:

script {
    display: none;
}

Extra

The <NOSCRIPT> tag is sometimes being used with the <SCRIPT> element for users that have disabled scripts in their browser, or have a browser that doesn't support client-side scripting.

The <SCRIPT> tag supports Global Attributes in HTML.

<!DOCTYPE html>
<html>
    <head>
        <title>web title</title>
        <script type="text/javascript">
            document.write("Hello World!");
        </script>
        <noscript>Your browser does not support JavaScript!</noscript>
     </head>
</html>

Differences Between HTML and XHTML

In XHTML, the content inside scripts is declared as #PCDATA (instead of CDATA), which means that entities will be parsed.

This means that in XHTML, all special characters should be encoded, or all content should be wrapped inside a CDATA section:

<!DOCTYPE html>
<html>
    <head>
        <title>web title</title>
        <script type="text/javascript">
            //<![CDATA[
                var i = 10;
                if (i < 5) {
                    // some code
                }
            //]]>
        </script>
     </head>
</html>
